## Overview

The Community Branch Plugin supports pull request decoration across multiple Application Lifecycle Management (ALM) platforms. Configure these integrations to enable automatic posting of SonarQube analysis results to your pull requests.

## Supported ALM Platforms

The plugin provides native support for the following platforms:

- **GitHub** (GitHub.com and GitHub Enterprise Server)
- **GitLab** (GitLab.com and self-hosted GitLab)
- **Bitbucket** (Bitbucket Server/Data Center and Bitbucket Cloud)
- **Azure DevOps** (Azure DevOps Services and Server)

## Configuration Overview

Each ALM platform requires specific configuration in SonarQube to enable pull request decoration:

1. **Server-level configuration**: Set up the ALM platform connection in SonarQube administration settings
2. **Project-level binding**: Link your SonarQube project to a repository in your ALM platform
3. **Analysis parameters**: Configure scanner parameters for pull request analysis

## Configuring ALM Bindings

ALM bindings are configured in SonarQube through the administration interface or via Web API.

### Available Actions

The plugin provides Web API actions for managing ALM bindings:

- `SetGithubBindingAction` - Configure GitHub integration
- `SetGitlabBindingAction` - Configure GitLab integration
- `SetBitbucketBindingAction` - Configure Bitbucket Server/Data Center integration
- `SetBitbucketCloudBindingAction` - Configure Bitbucket Cloud integration
- `SetAzureBindingAction` - Configure Azure DevOps integration
- `DeleteBindingAction` - Remove an ALM binding
- `ValidateBindingAction` - Validate an ALM binding configuration

### Validation

Each ALM platform has a dedicated validator to ensure proper configuration:

- `GithubValidator` - Validates GitHub connections and credentials
- `GitlabValidator` - Validates GitLab connections and credentials
- `BitbucketValidator` - Validates Bitbucket connections and credentials
- `AzureDevopsValidator` - Validates Azure DevOps connections and credentials

## Image Serving for PR Decoration

By default, images for pull request decoration are served as static resources from the SonarQube server.

<Warning>
  If your SonarQube server is behind a firewall and your ALM platform cannot access it, you must configure an external image base URL. See [Global Settings](/configuration/global-settings#image-base-url) for configuration details.
</Warning>

## Client Factories

The plugin uses dedicated HTTP client factories for each ALM platform:

- `GithubClientFactory` - Creates authenticated GitHub API clients
- `DefaultGitlabClientFactory` - Creates authenticated GitLab API clients
- `DefaultBitbucketClientFactory` - Creates authenticated Bitbucket API clients
- `DefaultAzureDevopsClientFactory` - Creates authenticated Azure DevOps API clients

These factories handle authentication, rate limiting, and API communication with each platform.
