## What is the Community Branch Plugin?

The SonarQube Community Branch Plugin extends the free Community Edition of SonarQube with powerful branch analysis and pull request decoration capabilities typically reserved for commercial editions. This plugin enables teams to analyze multiple branches, decorate pull requests with quality metrics, and integrate seamlessly with popular ALM platforms.

## Key Features

<AccordionGroup>
  <Accordion title="Branch Analysis in Community Edition" icon="code-branch">
    Analyze multiple branches of your project and track code quality trends over time. The plugin enables the `sonar.branch.name` parameter, allowing you to maintain separate quality profiles for different branches.
  </Accordion>

  <Accordion title="Pull Request Decoration" icon="comment-dots">
    Automatically decorate pull requests with:
    - Quality gate status (pass/fail)
    - New issues introduced in the PR
    - Code coverage changes
    - Inline comments on specific lines
  </Accordion>

  <Accordion title="Multi-Platform ALM Integration" icon="plug">
    Native integration with:
    - **GitHub**: Comments, status checks, and code annotations
    - **GitLab**: Merge request comments and discussions
    - **Azure DevOps**: Pull request threads and build status
    - **Bitbucket Cloud & Server**: PR comments and inline annotations
  </Accordion>

  <Accordion title="CI/CD Auto-Configuration" icon="robot">
    Automatic detection and configuration for:
    - GitHub Actions
    - GitLab CI
    - Azure Pipelines
    - Jenkins
    - Bitbucket Pipelines
    - Cirrus CI
    - CodeMagic
  </Accordion>

  <Accordion title="Flexible Deployment Options" icon="server">
    Deploy via:
    - Manual JAR installation
    - Official Docker image
    - Docker Compose
    - Kubernetes with Helm charts
  </Accordion>

  <Accordion title="Branch Retention Policies" icon="clock">
    Customize which branches are kept when inactive:

    - Protect specific branches from deletion
    - Regular expression patterns for branch names

## How It Works

The Community Branch Plugin works by:

1. **Java Agent Instrumentation**: Uses bytecode manipulation to extend SonarQube's functionality at runtime
2. **Web UI Enhancement**: Replaces frontend components to display branch and PR data
3. **Compute Engine Extension**: Adds post-analysis tasks to decorate pull requests
4. **Scanner Auto-Configuration**: Detects CI environment and automatically sets branch parameters
