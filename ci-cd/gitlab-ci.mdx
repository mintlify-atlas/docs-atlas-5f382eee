## Auto-Configuration

The plugin automatically detects GitLab CI environments and configures branch and merge request analysis without manual property configuration.

### Detection

The auto-configurer activates when:

- `GITLAB_CI=true` is set in the environment

### Environment Variables Used

| Variable | Purpose | Example |
|----------|---------|----------|
| `GITLAB_CI` | Detects GitLab CI environment | `true` |
| `CI_MERGE_REQUEST_IID` | Merge request ID | `42` |
| `CI_MERGE_REQUEST_SOURCE_BRANCH_NAME` | MR source branch | `feature/add-feature` |
| `CI_MERGE_REQUEST_TARGET_BRANCH_NAME` | MR target branch | `main` |
| `CI_COMMIT_REF_NAME` | Branch or tag name | `develop` |

## Configuration Behavior

### Merge Request Detection

The plugin detects merge requests when:

- `CI_MERGE_REQUEST_IID` is not empty

**Automatically configured properties:**

- `sonar.pullrequest.key` - Set to `CI_MERGE_REQUEST_IID`
- `sonar.pullrequest.branch` - Set to `CI_MERGE_REQUEST_SOURCE_BRANCH_NAME`
- `sonar.pullrequest.base` - Set to `CI_MERGE_REQUEST_TARGET_BRANCH_NAME`

### Branch Analysis

For non-MR builds (direct branch pushes):

**Automatically configured properties:**

- `sonar.branch.name` - Set to `CI_COMMIT_REF_NAME`

## Configuration File

Create a `sonar-project.properties` file in your repository root:

```properties
sonar.projectKey=my-project
sonar.sources=src
sonar.tests=tests
sonar.exclusions=**/node_modules/**,**/dist/**
sonar.coverage.exclusions=**/*.test.js,**/*.spec.js
```

## CI/CD Variables

Configure the following variables in GitLab at **Settings > CI/CD > Variables**:

| Variable | Value | Protected | Masked |
|----------|-------|-----------|--------|
| `SONAR_TOKEN` | Your SonarQube token | Yes | Yes |
| `SONAR_HOST_URL` | Your SonarQube server URL | No | No |

## Pipeline Configuration

### For Merge Requests Only

If you only want to analyze merge requests:

```yaml
sonarqube-check:
  image: sonarsource/sonar-scanner-cli:latest
  variables:
    SONAR_USER_HOME: "${CI_PROJECT_DIR}/.sonar"
    GIT_DEPTH: "0"
  cache:
    key: "${CI_JOB_NAME}"
    paths:
      - .sonar/cache
  script:
    - sonar-scanner
  rules:
    - if: $CI_PIPELINE_SOURCE == 'merge_request_event'
```

### For Branches and MRs

Analyze both protected branches and merge requests:

```yaml
sonarqube-check:
  image: sonarsource/sonar-scanner-cli:latest
  variables:
    SONAR_USER_HOME: "${CI_PROJECT_DIR}/.sonar"
    GIT_DEPTH: "0"
  cache:
    key: "${CI_JOB_NAME}"
    paths:
      - .sonar/cache
  script:
    - sonar-scanner
  rules:
    - if: $CI_PIPELINE_SOURCE == 'merge_request_event'
    - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH
    - if: $CI_COMMIT_BRANCH == 'develop'
```

## Advanced Configuration

### Quality Gate Integration

Fail the pipeline if quality gate fails:

```yaml
sonarqube-check:
  image: sonarsource/sonar-scanner-cli:latest
  variables:
    SONAR_USER_HOME: "${CI_PROJECT_DIR}/.sonar"
    GIT_DEPTH: "0"
  cache:
    key: "${CI_JOB_NAME}"
    paths:
      - .sonar/cache
  script:
    - sonar-scanner -Dsonar.qualitygate.wait=true
  allow_failure: false
  only:
    - merge_requests
    - main
```

### With Test Coverage

For Node.js projects with Jest:

```yaml
sonarqube-check:
  image: node:20-alpine
  variables:
    SONAR_USER_HOME: "${CI_PROJECT_DIR}/.sonar"
    GIT_DEPTH: "0"
  cache:
    key: "${CI_JOB_NAME}"
    paths:
      - .sonar/cache
      - node_modules/
  before_script:
    - npm ci
    - npm test -- --coverage
  script:
    - npx sonar-scanner
  only:
    - merge_requests
    - main
```

## Manual Override

If you need to override the auto-detected values:

```yaml
sonarqube-check:
  image: sonarsource/sonar-scanner-cli:latest
  script:
    - |
      sonar-scanner \
        -Dsonar.pullrequest.key=${CI_MERGE_REQUEST_IID} \
        -Dsonar.pullrequest.branch=${CI_MERGE_REQUEST_SOURCE_BRANCH_NAME} \
        -Dsonar.pullrequest.base=${CI_MERGE_REQUEST_TARGET_BRANCH_NAME}
```

## Important Notes

<Info>
**Shallow Clones**

Set `GIT_DEPTH: "0"` to disable shallow clones for accurate blame information.
</Info>

<Warning>
**Merge Request Pipelines**

Ensure merge request pipelines are enabled in **Settings > CI/CD > General pipelines** for MR decoration to work.
</Warning>

## Troubleshooting

### MR Decoration Not Working

1. Verify GitLab integration is configured in SonarQube
2. Check that the GitLab token has appropriate permissions
3. Ensure `sonar.core.serverBaseURL` is set in SonarQube settings
4. Verify merge request pipelines are enabled

### Environment Variables Not Set

Merge request variables are only available in merge request pipelines. Debug with:

```yaml
- name: Debug Environment
  script:
    - echo "GITLAB_CI=$GITLAB_CI"
    - echo "CI_MERGE_REQUEST_IID=$CI_MERGE_REQUEST_IID"
    - echo "CI_MERGE_REQUEST_SOURCE_BRANCH_NAME=$CI_MERGE_REQUEST_SOURCE_BRANCH_NAME"
    - echo "CI_MERGE_REQUEST_TARGET_BRANCH_NAME=$CI_MERGE_REQUEST_TARGET_BRANCH_NAME"
    - echo "CI_COMMIT_REF_NAME=$CI_COMMIT_REF_NAME"
```
